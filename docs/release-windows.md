# Windows release runbook

## Scope

Version `0.2.0` ships one unsigned, per-user NSIS installer for Windows. No code
signing, auto-update, or GitHub Release publishing occurs in this repository
workflow.

## Prerequisites

- Windows 10 22H2 or Windows 11 build machine.
- Node.js and npm compatible with the locked dependencies.
- Rust stable with the MSVC Windows target.
- Internet access only to restore declared build dependencies when not cached.

The release build does not need a Codex login, a local profile, DPAPI access,
usage logs, or a provider endpoint.

## Build

From the repository root:

```powershell
npm ci
npm test
npm run build
Push-Location src-tauri
cargo fmt --check
cargo test
cargo check
Pop-Location
npm run release:windows
npm run release:checksums
```

Expected output:

```text
release/QuotaBuddy_0.2.0_x64-setup.exe
release/QuotaBuddy_0.2.0_x64-setup.exe.sha256
```

The filename is generated by Tauri and can change with a version or target
change. The checksum script copies exactly one NSIS `.exe` from the Tauri
bundle directory and fails if that invariant is not met.

## Verify

```powershell
Get-FileHash .\release\QuotaBuddy_0.2.0_x64-setup.exe -Algorithm SHA256
Get-Content .\release\QuotaBuddy_0.2.0_x64-setup.exe.sha256
```

Compare the lowercase hash to the value before the `*` in the `.sha256` file.
Install only after the values match. Test installation on a non-production
Windows account; the installer is intentionally unsigned for this release.

## Installed upgrade gate

The release candidate is not final until the exact artifact from the target
`main` commit passes an installed upgrade and clean-install smoke test:

1. Record the `main` commit SHA, installer hash, installed file version,
   uninstall registry entry, autostart entry, Codex CLI resolution, and the hash
   of `%APPDATA%\com.quotabuddy.app\monitor-preferences.json`.
2. Back up that preferences file outside the QuotaBuddy install and app-data
   directories. Never select **Delete app data** during the upgrade test.
3. Close QuotaBuddy, install the new per-user NSIS package, and verify the
   installed version changed to `0.2.0` while the preference hash remained
   unchanged. Starting the executable twice must leave one process and focus
   the existing tray panel.
4. Validate the installed Tauri/WebView2 window, not a development server or
   browser mock. The panel must show real content rather than `about:blank`,
   open from the tray, refresh Codex usage, preserve autostart off, and keep the
   CLI available on `PATH`.
5. Exercise Desktop Acrylic on compatible Windows 11 with transparency on and
   the opaque fallback with transparency off. Windows 10 always uses the solid
   fallback; undocumented experimental Acrylic is intentionally unsupported.
6. For a clean-install check, retain the external backup, uninstall with app
   data deletion explicitly selected, verify QuotaBuddy-owned paths are gone,
   reinstall the same artifact, and restore the saved preferences if needed.
   The uninstaller must not modify the Codex profile or the user/machine PATH.

## Publish handoff

1. Download the artifact produced by the `main` workflow for the exact commit
   that will be tagged; do not substitute a pull-request or local build.
2. Review the CI artifact and its checksum.
3. Review `LICENSE`, `NOTICE`, and dependency changes.
4. Create the `v0.2.0` GitHub Release manually and attach both files unchanged.
5. Copy the commit SHA, SHA-256 value, unsigned/current-user status, Windows 10
   solid fallback, and absence of auto-update into the release notes.

Do not attach files or create a GitHub Release from CI.
